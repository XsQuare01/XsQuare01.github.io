# 글쓰기 규칙 — '바른 문장 쓰기'

이 블로그 포스트를 **쓰거나 고칠 때** 지키는 문장 규칙이다. 출처는 이슈 #65와 사용자 지침, 참고 자료 「'바른 문장 쓰기' 원칙」(원칙 1~12)이다.

**적용 시점**: 새 글을 쓸 때는 초안을 마친 뒤 이 체크리스트로 한 번 훑는다. 기존 글을 조판/수정할 때도 같은 PR에서 함께 통과시킨다. `review-post`의 L1 문체 검사도 이 문서를 기준으로 삼는다.

## 대원칙

- **과형식화·과교정 금지.** 의미·수식·코드·용어·직관 설명의 자연스러움은 건드리지 않는다. 문체만 조인다.
- **강박 금지.** 접속어를 0으로 만들거나 '것/의'를 전부 없애려 하지 않는다. 논리 전환에 꼭 필요한 것은 남긴다. 목적은 명료함이지 획일화가 아니다.
- 코드 주석·인용·수식 내부 텍스트는 대상이 아니다.

## 체크리스트

### 군더더기 없애기

- [ ] **접속어 최소화 (최우선).** 문두의 '그리고·그런데·그러나·따라서·하지만·그래서·그러니' 남발을 자제한다. 문장 순서·인과로 대체하거나 삭제한다. 접속어는 글의 긴장감을 떨어뜨린다. *(사용자 지침 + 원칙 2)*
- [ ] **보조 용언** 중 불필요한 것 제거. (예: "~해 보다", "~하고 있다"에서 군더더기) *(원칙 1)*
- [ ] **불필요한 수식어**(관형어·부사어) 제거. *(원칙 3)*
- [ ] **접미사 '-들'** 등 문맥상 없어도 되는 복수 표지 생략. *(원칙 4)*
- [ ] **지시어**('이·그·이러한')의 대상이 분명하고 적합한지 점검. *(원칙 5)*

### 중복 없애기

- [ ] **동어 중복** — 같은 단어 반복(의도적 강조가 아니면) 정리. *(원칙 6)*
- [ ] **동의 중복(겹말)** — 같은 뜻이 겹치는 표현 제거. (예: '미리 예상', '역전 앞', '남은 여생') *(원칙 7·8)*
- [ ] **같은 문법 요소 중복** — 조사·어미의 반복으로 흐름이 끊기지 않게 정리. *(원칙 9)*

### 간결·우리말

- [ ] **의존 명사 '것' 줄이기.** '~하는 것이다/~라는 것' 남용을 평서문으로. *(원칙 10)*
- [ ] **불필요한 표현** 제거로 간결화. *(원칙 11)*
- [ ] **관형격 조사 '의' 줄이기.** *(원칙 12)*

## 검출 보조 (참고용, 삭제는 문맥 판단)

편별로 후보를 기계적으로 셀 때 쓰는 표지. 수치는 참고일 뿐, 위반 여부는 문맥으로 판단한다.

```bash
# 문두 접속어
grep -oE '(^|[.」”"] ?)(하지만|그리고|그런데|따라서|그러나|그래서|그러니|그러면)' <파일> | wc -l
# 의존명사 '것'
grep -oE '것(이다|이|을|은|과|도|만|의|들)?' <파일> | wc -l
# 복수 '-들'
grep -oE '[가-힣]들' <파일> | wc -l
```

## 완료 기준 (편당)

- 문두 접속어가 꼭 필요한 곳에만 남는다.
- 겹말·군더더기·불필요한 '것/의/-들'이 줄어 문장이 짧고 또렷하다.
- 의미·수식·코드는 그대로다. 직관 설명의 자연스러움을 해치지 않는다.
- `npm run build` 통과.
