3. **## 구간 스케줄링 문제**
   - $N$개의 일 $J_i = (S_i, T_i)$. $S_i$=시작 시간, $T_i$=종료 시간.
   - 이익은 모두 동일 → 목표는 **겹치지 않게 고른 일의 개수**를 최대화.
   - "겹친다"의 정의: 두 구간의 시간이 교차하면(한쪽 종료 전에 다른 쪽 시작) 함께 할 수 없다.
   - 1편(이익 합 최대화, 슬롯 배치)과의 차이를 짧게 언급.
   - 작은 예시 + 최적해.
4. **## Greedy 전략** — **종료 시간 오름차순 정렬 → 가장 빨리 끝나는 일부터 본다. 이미 고른 일과 겹치면 버린다.** 왜 "시작 시간"이나 "길이"가 아니라 "종료 시간"이 기준인지 직관 설명(빨리 끝낼수록 뒤에 남는 시간이 많다). 이 자리에 SVG 삽입.
5. **## 정확성 증명** — 교환 논증.
   - **Invariant**: 종료 시간 순으로 $i$번째까지 처리한 greedy 상태를 $A_i$ 라 할 때, $A_i$의 모든 결정과 일치하는 최적해 $S$ 가 적어도 하나 존재한다 (즉 $j \le i$ 인 $J_j$ 의 채택/버림 여부가 $A_i$와 $S$에서 같다).
   - **Base**: $i = 0$, vacuously true.
   - **Step**: $J_{i+1}$ 을 처리할 때.
     - **경우 1 — $A_i$가 $J_{i+1}$을 버림.** 이미 고른 일과 겹쳐서 버린 경우. 그 겹치는 일은 $j \le i$ 이고 $A_i = S$ 범위에 있으므로 $S$에도 있다. 따라서 $S$도 $J_{i+1}$ 을 가질 수 없다 → 둘 다 버림, 불변식 유지.
     - **경우 2 — $A_i$가 $J_{i+1}$을 채택했는데 $S$엔 없음.** $S$에서 $J_{i+1}$ 과 충돌하는 일 중 종료 시간이 가장 빠른 $J_x$ 를 본다. 종료 시간 오름차순이므로 $x > i+1$ → $T_{i+1} \le T_x$. $S$에서 $J_x$ 를 $J_{i+1}$ 로 **교환**해도 (a) 개수는 그대로이고 (b) 새 충돌이 생기지 않음을 보인다.
       - $J_{i+1}$ 의 시작 $S_{i+1}$ 과 $J_x$ 의 시작 $S_x$ 관계로 무충돌 확인:
         - $S_x \le S_{i+1}$ 이면 $S_x \le S_{i+1} < T_{i+1} \le T_x$ 이라 $J_{i+1}$ 은 $J_x$ 가 차지하던 구간 안에 들어가므로 다른 일과 겹치지 않는다.
         - $S_x > S_{i+1}$ 이면 $J_{i+1}$ 의 시작이 더 앞이다. 만약 그 앞에 겹치는 다른 일 $J_y$ 가 $S$에 있었다면 $T_y \le S_x$ 인데, 그러면 $T_y < T_{i+1}$ 이 되어 종료 시간이 $J_{i+1}$ 보다 빠른 일이 $S$에 있는 셈이라 greedy/정렬 전제와 모순. 따라서 그런 $J_y$ 는 없고 교환해도 겹치지 않는다.
       - 교환 결과 $S$ 는 여전히 최적(개수 동일)이고 $J_{i+1}$ 을 가진다 → 불변식 유지.
   - 증명 마무리는 callout으로 요약.
