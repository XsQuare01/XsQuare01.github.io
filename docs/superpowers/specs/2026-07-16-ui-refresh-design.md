# 블로그 UI 리프레시 — 디자인 스펙

- 날짜: 2026-07-16
- 브랜치: `design/ui-refresh`
- 상태: 설계 확정 대기 (사용자 리뷰 전)

---

## 주장 (Design Thesis)

> **기계는 터미널로 말하고, 사람은 논문으로 읽는다.**

이 블로그는 평면 위 기하 알고리즘을 다루는 장문 + 증명 + 손그림 다이어그램의 공간이다.
디자인의 모든 결정은 하나의 구분에서 흘러나온다:

- **사람이 읽는 것(산문)** → 조판된 논문. 종이색, 세리프, 넓은 여백, 정리/증명 블록.
- **기계가 내놓는 것(네비게이션·메타데이터·코드·다이어그램)** → 터미널. 다크 표면, 모노스페이스, 프롬프트, 앰버 인광색.

두 세계는 폰트가 아니라 **공간으로 분리**되므로 충돌하지 않는다. 앰버 액센트 하나가 둘을 관통해 하나의 시스템으로 묶는다.

이 주장은 콘텐츠 자체(알고리즘·기하)에서 나오므로 "AI 기본값"과 반대다. AI 기본값에는 주장이 없다.

---

## 걷어내는 것 (현재 CSS의 "AI 티")

현 `global.css`에서 제거·대체한다:

- 3색 채도 액센트(Tailwind orange-500 / sky-400 / violet-400) → **앰버/테라코타 단색 액센트**
- hover 글로우 box-shadow → 제거 (또는 정적인 미세 그림자만)
- `radial-gradient` 배경 blob (오렌지·하늘색 방사) → 제거, 종이/다크 단색 표면
- h2 앞 gradient accent bar → **모노스페이스 `##` 접두사**로 대체
- staggered `fadeInUp` + `cubic-bezier(0.25,0.46,0.45,0.94)` 통일 모션 → 절제된 전환만 유지 (과한 등장 애니메이션 제거)

**지키는 것:** 고정 사이드바 레이아웃(개인 블로그 정체성), Pretendard(한국어 산세리프, 단 본문 세리프와 역할 분리), 15px/1.7 본문 리듬.

---

## 테마 (확정)

- **다크 크롬 + 라이트 본문** 고정 이중 톤.
  - 크롬(사이드바·코드·다이어그램 플레이트) = 다크 터미널.
  - 본문(읽는 영역) = 오프화이트 종이.
- **기존 light/dark 토글 제거** — 이제 테마는 "영역별 고정"이지 "사용자 전환"이 아니다.
  (※ 이 결정은 사용자 리뷰에서 재확인. 토글 유지를 원하면 다크-본문 변형을 별도 설계해야 함.)

## 색 토큰

```
/* 종이 (본문) */
--paper:      #faf8f3;   /* 오프화이트 종이 */
--ink:        #22201b;   /* 본문 잉크 */
--ink-soft:   #5c574d;   /* 보조 텍스트 */
--ink-faint:  #918a7b;   /* 캡션·여백노트·byline */
--rule:       #e2dccf;   /* 종이 위 구분선 */

/* 액센트 (앰버/테라코타, 종이·다크 양쪽에서 동작) */
--amber:      #b56a1a;   /* 종이 위 액센트 */
--amber-soft: rgba(181,106,26,.10);  /* 정리 블록 배경 */
--term-amber: #d99b4e;   /* 다크 위 액센트(밝게) */

/* 터미널 크롬 */
--term-bg:    #15171c;
--term-fg:    #cdd3cc;
--term-dim:   #6a7069;
--term-green: #6fc07a;   /* 프롬프트 user@host */

/* 폰트 */
--serif: 'Iowan Old Style','Palatino Linotype',Georgia,serif;  /* 본문 산문 */
--mono:  'JetBrains Mono','Cascadia Code',monospace;           /* 크롬·코드·메타 */
--sans:  'Pretendard', -apple-system, sans-serif;              /* UI 라벨 등 */
```

> 웹폰트: 본문 세리프는 한글 지원이 관건. `Iowan/Palatino`는 라틴 전용이므로 **한글 세리프 웹폰트 선정 필요**(예: Nanum Myeongjo, RIDIBatang 등). 이는 플랜 단계에서 성능(용량)과 함께 결정한다. 잠정: 한글은 명조 계열, 라틴/숫자/수식은 세리프.

## 레이아웃 & 컴포넌트

### 1) 사이드바 (터미널)
- 다크(`--term-bg`), 모노스페이스.
- 상단: `xsquare@blog` (user=green, host=dim), `~/algorithms`, `N posts · Mk views`.
- 네비: 파일 경로 스타일. 활성 항목 `▸` 접두 + 앰버.
- 하단: 깜빡이는 블록 커서 프롬프트(`$ ▍`) — 정적/최소 모션.

### 2) 아티클 헤더
- 제목 위 **커맨드 에코**: `$ cat series/convex-hull/05.md | render` (mono, dim, 앰버 `$`/`|`).
- 키커: mono 대문자 + letter-spacing, 앰버 (`COMPUTATIONAL GEOMETRY · 05`).
- 제목: 세리프, 2rem, `letter-spacing:-.015em`.
- byline: 세리프 이탤릭, `--ink-faint`.

### 3) 섹션 헤딩
- 세리프. 앞에 mono 앰버 `##` 접두사(작게, 400 weight). 기존 gradient bar 대체.

### 4) 정리/콜아웃 블록 (`.thm` / 기존 `.callout`)
- 왼쪽 앰버 룰 3px + `--amber-soft` 배경.
- 라벨(`THEOREM`, `이 포스트에서 다루는 내용` 등): mono 대문자 앰버, 블록 상단.
- 본문: 세리프. 증명 끝 `∎` 허용.

### 5) 여백 노트 (margin note) — 신규
- 데스크톱: 본문 오른쪽 여백에 절대배치(mono, faint, 상단 룰).
- ≤1080px: 인라인으로 접힘(왼쪽 룰 보더).
- 용도: "어려운 곁가지"를 본문 흐름을 끊지 않고 옆에 다는 자리. (별도 '추가 설명' 페이지 규칙과 병행 — 짧은 주석은 margin note, 큰 곁가지는 별도 페이지.)

### 6) 코드 블록 (두 세계의 접점)
- 종이 위에 박히는 **다크 터미널 카드**: `--term-bg`, mono, 라운드, 미세 그림자.
- 파일명 라벨(mono dim 대문자), 주석=dim, 키워드=term-amber, 함수=term-green.
- 언어: C++ 우선(기존 방침), pseudocode 허용.

### 7) 다이어그램 = 다크 플레이트 (SVG 마이그레이션 0)
- 기존 156개 SVG는 다크 배경 기준(baked-in dark bg + 밝은 텍스트).
- 따라서 본문 이미지/`figure`를 **다크 매트(플레이트)** 위에 얹는다: 라운드, 패딩, `--term-bg` 계열 배경, 캡션은 종이 위 `--ink-faint`.
- 결과: 기존 SVG를 **한 개도 고치지 않고** "기계의 출력물" 규칙에 자연스럽게 편입. 다이어그램·코드·크롬이 모두 다크로 일관.
- 신규 다이어그램도 동일 규칙(다크 배경 유지)로 계속 제작.

### 8) 링크·태그
- 링크: 종이 위 앰버, hover 밑줄(글로우 아님).
- 태그: mono 소형 pill, 앰버 보더/텍스트, hover 미세 배경.

## 반응형

- 사이드바: 데스크톱 고정 → 모바일 상단 접이식(햄버거) 또는 상단 바.
- 본문 컬럼: `max-width` 유지(약 660px), 좌우 패딩.
- margin note: ≤1080px 인라인 접힘.
- 코드·다이어그램 카드: `overflow-x:auto`로 가로 스크롤 격리(본문 가로 스크롤 금지).

---

## 접근성

- 대비: 종이/잉크(#faf8f3 / #22201b) 및 다크/term-fg 모두 WCAG AA 이상 확인.
- 앰버 액센트가 유일 액센트이므로 색만으로 정보 전달 금지(아이콘/텍스트 병행).
- `:focus-visible` 앰버 아웃라인 유지.
- `prefers-reduced-motion`: 커서 깜빡임·전환 비활성.

---

## 영향 파일 (예상)

- `src/styles/global.css` — 토큰·컴포넌트 전면 개편(주 작업).
- `src/layouts/BaseLayout.astro` — 사이드바(터미널), 테마 토글 제거.
- `src/layouts/PostLayout.astro` — 아티클 헤더(커맨드 에코·키커), 섹션 헤딩, figure 플레이트.
- 포스트 콘텐츠(`src/content/posts/*.md`) — **변경 없음**(마크다운 그대로, 스타일만 바뀜).
- SVG(`public/images/**`) — **변경 없음**.
- 폰트: 한글 세리프 웹폰트 추가(플랜에서 선정) — `public/` 또는 CDN, 성능 고려.

## 범위 밖 (Non-goals)

- 포스트 내용/문장 수정, SVG 재작업.
- 새 페이지/기능 추가(검색, 댓글 등).
- 라이트/다크 사용자 토글 재도입(이번 스펙은 고정 이중 톤).
