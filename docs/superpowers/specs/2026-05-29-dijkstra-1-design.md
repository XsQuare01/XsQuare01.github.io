    - 정확성 증명 2가지

### 3.2. 알고리즘의 아이디어

- 입력 정의: $G = (V, E, w)$, 정점 수 $n$, 시작점 $v_0$
- 도착점을 지정하지 않는 이유 한 단락 — 임의의 $v_n$의 최단 경로 위에 있는 모든 중간 정점의 최단 거리가 동시에 얻어지므로 (노션의 회색 주석을 그대로 가져옴)
- $d_{min}(u)$의 초기값 정의:
    - $v_0$와 $u$ 사이에 엣지가 있으면 $w(v_0, u)$
    - 없으면 $\infty$
- 답 집합 $R$ 도입, 시작 시 $R = \{v_0\}$
- 모든 정점이 $R$에 포함될 때까지 다음 3-step 반복:
    1. $R^C$ 중 가장 작은 $d_{min}(u)$를 가진 $u$를 고른다.
    2. $u$를 $R$에 추가한다.
    3. $R^C$의 모든 $v$에 대해 $d_{min}(v) = \min(d_{min}(v), d_{min}(u) + w(u, v))$로 갱신한다.

### 3.3. Pseudo-code

`prim.md`와 동일한 `cpp` 코드 블록 스타일.

```cpp
// G = (V, E, w), 시작 정점 v0
R       = { v0 }
d[v0]   = 0
d[v]    = ∞                    for all v ≠ v0
d[v]    = w(v0, v)             if (v0, v) ∈ E

while (R != V) {
    u = argmin { d[u] : u ∉ R };
    R = R ∪ { u };
    for each (u, v) ∈ E with v ∉ R {
        d[v] = min(d[v], d[u] + w(u, v));
    }
}
return d;
```

- 한 단락으로 "한 번의 반복에 정확히 정점 한 개가 $R$에 추가되므로 총 $n-1$번 반복" 설명
- 그래프 가정 한 줄 각주: 엣지 가중치 $\ge 0$, $v_0$에서 모든 정점에 도달 가능

### 3.4. 단계별 시각화

- 정점 5개짜리 작은 그래프 한 개로 4단계 진행을 그림(`progress.svg`)으로 보여준다.
- **예시 그래프 정의** (그림과 표가 같은 그래프를 가리키도록 여기서 못박는다):
    - 정점 $\{A, B, C, D, E\}$, 시작점 $A$
    - 엣지: $A{-}B(2)$, $A{-}C(5)$, $B{-}C(1)$, $B{-}D(7)$, $C{-}D(3)$, $C{-}E(8)$, $D{-}E(2)$
    - 의도된 추가 순서: $A \to B(d=2) \to C(d=3) \to D(d=6) \to E(d=8)$ — 모든 단계에서 갱신이 한 번씩 일어나도록 가중치 설계
- 그림 바로 아래 단계별 표 (위 그래프 기준):

    | 반복 | 선택된 $u$ | $R$ (추가 후) | 갱신된 $d_{min}$ 값들 |
    |---|---|---|---|
    | 1 | $B$ | $\{A, B\}$ | $C: 5 \to 3$, $D: \infty \to 9$ |
    | 2 | $C$ | $\{A, B, C\}$ | $D: 9 \to 6$, $E: \infty \to 11$ |
    | 3 | $D$ | $\{A, B, C, D\}$ | $E: 11 \to 8$ |
    | 4 | $E$ | $\{A, B, C, D, E\}$ | — |

- 표 다음 한 줄: "이 그림이 시사하듯이, Dijkstra는 항상 **가까운 노드부터** 추가한다." 정식 증명은 짧은 각주로 2편 예고만 한다 (별도 콜아웃 만들지 않음).

### 3.5. 정확성 증명

`prim.md`의 "증명 → 경우 1/경우 2 → 표" 구조를 차용하지만, 다익스트라에는 cycle 논증 대신 노션의 **두 sub-proof**를 옮긴다.

#### 3.5.1. 가장 작은 $d_{min}(u)$가 정말 $u$의 최단거리인가

- 가정: $u$가 $R^C$ 중 가장 작은 $d_{min}$을 가졌는데 그것이 $u$의 최단거리가 **아니라**고 가정.
- 그렇다면 $R$만 거치는 길이 $d_{min}(u)$ 외에 더 짧은 다른 경로가 있어야 한다. 그 경로는 어느 순간 반드시 $R$에서 $R^C$로 빠져나오는 첫 정점 $u'$를 지난다.
- $u'$ 역시 $R^C$의 원소이므로 $d_{min}(u) \le d_{min}(u')$.
- 그 경로의 길이는 $u'$에 도달한 시점에 이미 $d_{min}(u')$ 이상 → 결국 $d_{min}(u)$ 이상 → 더 짧을 수 없음 → 모순.
- **표 1개**로 정리 (가정 / 관찰 / 결론).
- 보조 그림 `proof-shortcut.svg`로 두 경로(직진/우회) 비교.

#### 3.5.2. 왜 새로 추가된 $u$의 엣지만 갱신해도 충분한가

- 의문: $u$가 $R$에 추가된 뒤, $u$를 거치고 다시 $R$의 다른 노드 $t$를 거쳐 $R^C$로 빠지는 경로는 왜 고려하지 않아도 되는가?
- 답: $t$는 $u$보다 먼저 $R$에 들어왔으므로, $u$를 거치지 않는 $d_{min}(t)$가 이미 확정되어 있다. $t$에 도달하는 더 짧은 길이 없음 → $u$를 우회해 $t$를 다시 들를 이유 없음.
- 따라서 $u$로부터 직접 나가는 엣지만 갱신하면 된다.
- 보조 그림 `proof-detour.svg`로 "$u \to t \to R^C$ 우회 경로가 불필요"를 시각화.

### 3.6. 핵심 정리 콜아웃 (`callout-key`)

`prim.md`의 "핵심 정리" 톤에 맞춰 4-5개 bullet:

- Dijkstra는 답 집합 $R$을 비워둔 채 시작점만 넣고, 매 단계 $R^C$ 중 가장 작은 $d_{min}$을 가진 정점을 골라 $R$에 추가한다.
- 한 번의 반복에 정점 하나가 확정되므로 총 $n-1$번 반복 후 종료.
- 정확성 1: 가장 작은 $d_{min}(u)$는 이미 $u$의 최단거리다 — $R^C$를 한 번이라도 들렀다 오는 경로는 더 짧을 수 없기 때문.
- 정확성 2: 새로 들어온 $u$로부터 나가는 엣지만 갱신해도 충분하다 — $R$ 안의 어떤 노드를 우회해도 더 짧아지지 않으므로.
- 결과적으로 시작점 하나에서 **모든 정점까지의 최단 거리**가 한 번에 얻어진다.

### 3.7. 증명의 묘미 콜아웃 (`callout-simple`)

한 단락. Prim과의 대비:

- Prim은 cycle 위의 대체 간선 $e'$과 비교해 그리디 선택이 항상 어떤 MST에 들어감을 보였다.
- Dijkstra는 "이미 확정된 거리"라는 사실 자체에 기댄다 — 새로 고른 $u$의 $d_{min}(u)$를 흔들 수 있는 경로는 반드시 $R^C$를 거쳐야 하고, $R^C$의 모든 노드는 정의상 $d_{min}(u)$ 이상이다.
- 같은 그리디 전략이지만 정당화 방식이 다르다.
