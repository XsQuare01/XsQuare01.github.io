````markdown
## 알고리즘의 아이디어

입력은 가중 그래프 $G = (V, E, w)$와 시작점 $v_0$, 정점 수는 $n$이다. **도착점은 따로 정하지 않는다.** 임의의 정점 $v_n$까지의 최단 경로를 구하면, 그 경로 위의 모든 중간 정점까지의 최단 거리도 동시에 얻어지기 때문이다 — 만약 중간의 어떤 부분 경로가 최단이 아니라면, 그것을 더 짧은 경로로 바꿔치기해 전체 길이를 줄일 수 있어 모순이 생긴다. 그래서 한 번에 모든 정점까지의 거리를 함께 구한다.

각 정점 $u$에 대해 **현재까지 추정한 최단 거리** $d_{min}(u)$를 관리한다. 초기값은 다음과 같다.

- 엣지 $(v_0, u)$가 존재하면: $d_{min}(u) = w(v_0, u)$
- 엣지가 없으면: $d_{min}(u) = \infty$

그리고 **이미 답이 확정된 정점들의 집합** $R$을 도입한다. 처음에는 $R = \{v_0\}$이다 (시작점까지의 거리는 자명히 $0$).

이제 $R$이 모든 정점을 포함할 때까지 다음 3-step을 반복한다.

1. $R^C$ (아직 확정되지 않은 정점들) 중 **가장 작은 $d_{min}$을 가진 정점** $u$를 고른다.
2. $u$를 $R$에 추가한다 — 이 시점에 $d_{min}(u)$는 $u$의 진짜 최단거리로 확정된다 (왜 그런지는 아래 [정확성 증명](#정확성-증명)에서).
3. $u$에서 나가는 엣지로 $R^C$의 인접 정점 $v$들에 대해 $d_{min}(v) = \min(d_{min}(v),\ d_{min}(u) + w(u, v))$로 갱신한다.

---

## Pseudo-code

```cpp
// G = (V, E, w), 시작 정점 v0
R       = { v0 }
d[v0]   = 0
d[v]    = ∞                    for all v ≠ v0
d[v]    = w(v0, v)             if (v0, v) ∈ E

while (R != V) {
    u = argmin { d[u] : u ∉ R };
    R = R ∪ { u };
    for each (u, v) ∈ E with v ∉ R {
        d[v] = min(d[v], d[u] + w(u, v));
    }
}
return d;
```

한 번의 반복은 **정점 하나를 $R$에 추가**한다. 시작 시 $|R| = 1$이고 끝났을 때 $|R| = n$이므로, 반복은 정확히 $n - 1$번 일어난다.

이 글은 다음 두 가정 위에서 동작한다 — (i) 모든 엣지 가중치는 $0$ 이상, (ii) 시작점 $v_0$에서 모든 정점에 도달 가능. 음의 가중치/도달 불가 정점 처리는 후속 글에서 다룬다.
````

````markdown

---

## 정확성 증명

Dijkstra가 매 단계 올바른 정점을 $R$에 추가한다는 것, 그리고 그때 갱신하는 엣지만으로 모든 거리가 정확히 유지된다는 것을 두 명제로 나누어 증명한다.

### 1. 가장 작은 $d_{min}(u)$가 정말 $u$의 최단거리인가

알고리즘은 $R^C$ 중 $d_{min}$이 가장 작은 정점 $u$를 골라 $R$에 추가하면서 "$d_{min}(u)$가 $u$의 최단거리"라고 선언한다. 이게 왜 맞을까?

**가정** — $d_{min}(u)$가 $u$의 최단거리가 **아니라**고 하자. 그러면 더 짧은 어떤 경로 $P$가 존재한다. 현재 $d_{min}(u)$는 "$R$의 노드들만 거쳐서 $u$에 도달하는 최단 길이"이므로, $P$는 반드시 어느 순간 $R$ 바깥으로 빠져나간다. 그 순간 처음으로 $R^C$에 들어가는 정점을 $u'$이라 하자.

![증명 1 — R^C를 거치는 우회 경로는 u'를 지나며 더 짧아질 수 없다](/images/dijkstra-1/proof-shortcut.svg)

**관찰** — $u'$ 역시 $R^C$의 원소이므로, $u$의 선택 조건에 의해

$$d_{min}(u) \le d_{min}(u')$$

이다. 그리고 $u'$에 도달하는 시점에 경로 $P$의 누적 길이는 이미 $d_{min}(u')$ 이상이다 ($d_{min}(u')$의 정의가 "$R$만 거쳐 $u'$에 도달하는 최단 길이"이므로).

**결론** — 따라서 $P$의 전체 길이는

$$|P| \ge d_{min}(u') \ge d_{min}(u)$$

가 되어, $P$가 $d_{min}(u)$보다 짧다는 가정에 **모순**. 따라서 $d_{min}(u)$는 $u$의 최단거리다.

| | 주장 |
|---|---|
| 가정 | $d_{min}(u)$가 $u$의 최단거리가 아니다 → 더 짧은 경로 $P$ 존재 |
| 관찰 | $P$는 어느 순간 $R^C$로 빠짐 → 첫 진입점 $u' \in R^C$ 존재 |
| 관찰 | $u$는 $R^C$ 중 가장 작은 $d_{min}$을 가지므로 $d_{min}(u) \le d_{min}(u')$ |
| 결론 | $\|P\| \ge d_{min}(u') \ge d_{min}(u)$ → 모순 |
````

````markdown

### 2. 왜 새로 추가된 $u$의 엣지만 갱신해도 충분한가

알고리즘은 $u$를 $R$에 추가한 직후, **$u$로부터 직접 나가는 엣지**만 사용해 $R^C$의 거리를 갱신한다. $R$ 안의 다른 정점들에 대해서는 아무것도 다시 보지 않는다. 이게 왜 충분할까?

**의문** — $u$를 거치고 다시 $R$의 어떤 정점 $t$를 들렀다가 $R^C$의 정점 $v$로 빠지는 경로는 왜 고려하지 않아도 되는가?

![증명 2 — u → t → R^C 우회 경로가 왜 불필요한지](/images/dijkstra-1/proof-detour.svg)

**관찰** — $t$는 $u$보다 **먼저** $R$에 들어온 정점이다 (그 시점에 $t$의 거리가 확정되었기 때문에 $R$에 있는 것). 그 시점에 결정된 $d_{min}(t)$는 $u$를 거치지 않는 경로의 길이다. 따라서

$$d_{min}(t) \le \underbrace{d_{min}(u) + w(u, t)}_{u \text{를 거쳐 } t \text{로 가는 경로 길이}}$$

가 성립한다 (그렇지 않다면 $t$의 거리가 더 줄어들 수 있었으므로 $d_{min}(t)$가 진짜 최단거리라는 사실에 모순).

**결론** — 그러므로 $u \to t \to v$ 경로의 길이는

$$d_{min}(u) + w(u, t) + w(t, v) \ge d_{min}(t) + w(t, v)$$

인데, **$d_{min}(t) + w(t, v)$**는 $t$가 $R$에 들어왔을 때 이미 $v$에 대해 갱신된 값이다. 즉 새로운 정보가 아니다. 따라서 $u$를 거쳐 $t$를 우회하는 경로는 무시해도 된다 — **$u$로부터 직접 $v$로 가는 엣지**만 새로 갱신하면 충분하다.
````

````markdown
<div class="callout callout-key">
<div class="callout-title">핵심 정리</div>

- Dijkstra는 답이 확정된 정점 집합 $R$을 시작점만 넣고 시작해, 매 단계 $R^C$ 중 **$d_{min}$이 가장 작은 정점**을 골라 $R$에 추가한다.
- 한 번의 반복이 정점 하나를 확정하므로 정확히 $n - 1$번 반복한 뒤 종료한다.
- **정확성 1** — 가장 작은 $d_{min}(u)$는 곧 $u$의 최단거리다. $R^C$를 한 번이라도 거치는 우회 경로는 $d_{min}(u')$ 이상이고, $d_{min}(u) \le d_{min}(u')$이므로 더 짧을 수 없기 때문.
- **정확성 2** — 새로 추가한 $u$로부터 직접 나가는 엣지만 갱신해도 된다. $R$ 안의 다른 노드 $t$를 우회하는 경로는 $t$가 이미 가진 $d_{min}(t)$가 최단인 이상 새로운 정보가 아니다.
- 결과적으로 한 시작점에서 **모든 정점까지의 최단 거리**가 단 한 번의 실행으로 얻어진다.
````
