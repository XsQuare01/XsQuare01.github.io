# main 디자인 재평가 — "주장하는 종이"

> 최신 `main`에 머지·배포된 블로그를 `docs/design/DESIGN_DIRECTION.md`의 주장과 동일한 평가 기준으로 검토한 기록이다.
> 이 문서는 디자인 판단의 근거와 다음 재평가의 기준선을 보존한다.

## 0. 결론

**종합 점수: 6.8/10**

이전 배포본의 6.2/10에서 0.6점 상승했다. 종이색, 잉크블루, 세리프 본문, 조용한 레일이 결합하면서 사이트의 첫인상은 개발자 대시보드에서 편집된 학습 노트로 분명히 이동했다.

다만 현재 상태는 다음과 같이 요약된다.

> **"엄밀한 학습 기록처럼 보이는 블로그"에는 도달했지만, "논증 구조 자체가 디자인이 된 블로그"에는 아직 도달하지 못했다.**

가장 큰 남은 문제는 무작위 태그 색, 별점 난이도, 카테고리·태그의 복제 카드 구조, 모바일 목차 부재, 작은 mono 한글 UI 라벨이다.

## 2. 실측 근거

### 2.1 홈

- 접힌 레일 폭: 약 `78.85px`
- 메인 콘텐츠 폭: `614px`
- 헤드라인: `Noto Serif KR`, `30px`, `800`, `36px` line-height
- 추천 글 카드: 약 `301.38px` 폭, `10px` radius
- 추천 글 카드에는 잉크블루 왼쪽 괘선과 장식용 원이 남아 있다.
- 최근 글은 카드보다 열린 피드 행에 가까워 추천 글과 위계가 구분된다.
- 주제별 현황은 잉크블루 단색 체계 안에서 조용하게 표시된다.

이전 배포본의 같은 중간 폭에서 메인 콘텐츠가 약 `432px`였던 것과 비교하면 조용한 레일 도입으로 읽기 영역이 실질적으로 넓어졌다.

### 2.2 카테고리 색인

- 메인 콘텐츠 폭: `614px`
- 활성 카테고리: 3개
- 카드 폭: 약 `301.38px`
- 카드 배경: 종이보다 한 단계 어두운 `rgb(241, 236, 224)`
- 카드 radius: `10px`
- 카드 3개가 2열로 배치되어 마지막 행 오른쪽이 비어 있다.

색상은 새 방향과 맞지만, 정보 구조는 여전히 범용 카드 그리드다. 넓은 오른쪽 여백과 불완전한 마지막 행은 의도적 편집 여백보다 미완성 레이아웃처럼 보일 수 있다.

### 2.3 태그 색인

- 태그 카드: 41개
- 2열 카드 그리드
- 첫 카드 폭: 약 `301.38px`
- 카드 radius: `10px`
- 전체 페이지 높이: 약 `2585px`

41개의 동일한 카드가 반복되어 논문 색인보다 자동 생성된 컴포넌트 목록처럼 보인다. 카테고리와 태그가 서로 다른 탐색 개념인데도 거의 같은 시각 문법을 쓴다.

### 2.4 글 상세 — 데스크톱

대상: `closest-pair-3`

- 글 본문 폭: `614px`
- 제목: `30px / 35.4px`
- 본문: `16.2px / 29.16px`
- 데스크톱 문서 높이: 약 `8035px`
- 우측 TOC: `240px`, sticky
- 코드 블록: 어두운 고대비 배경, 자체 가로 스크롤
- 표와 수식: 화면을 깨뜨리지 않고 가로 스크롤 가능
- `numbered: true`에 따라 섹션 번호가 잉크블루로 표시된다.
- 콜아웃은 잉크블루 요점, 회색 보충, 적색 경고 중심으로 정리됐다.

섹션 번호와 단색 콜아웃은 디자인 방향을 실제 화면에서 보여 주는 개선이다. 다만 `.thm`, `.proof`, `.qed` 같은 정리·증명 환경은 CSS에 존재하지만 실제 글에서 폭넓게 사용되지 않아 점수에는 제한적으로만 반영했다.

### 2.5 글 상세 — 모바일

- 본문 폭: 약 `345.4px`
- 제목: `21px / 24.78px`
- 본문: `16.2px / 29.16px`
- 전체 문서 높이: 약 `10467px`
- 가로 overflow: 없음

본문 가독성과 overflow 처리는 안정적이다. 그러나 1만 px가 넘는 장문에서 목차가 완전히 사라져 현재 위치와 전체 구조를 확인하기 어렵다.

## 4. 항목별 판단

### 4.1 의미에 따른 시각적 구분 — 6.8/10

#### 잘된 점

- 콜아웃이 요점, 보충, 경고의 3개 계열로 정리됐다.
- `closest-pair-3`의 섹션 번호는 논문·교재의 구조와 연결된다.
- 제목, 본문, 메타데이터, 코드, 목차의 위계가 색 없이도 비교적 잘 읽힌다.
- 활성 상태와 주요 강조가 잉크블루로 모였다.

#### 감점 근거

- `src/utils/tagColor.ts`의 hash 기반 다색 태그가 남아 있다.
- `src/utils/difficulty.ts`의 별점 난이도가 그대로 사용된다.
- 카테고리와 태그 색인이 거의 같은 2열 카드 그리드를 사용한다.
- 정리·증명 문법은 CSS 능력으로 존재하지만 현재 콘텐츠의 일반적인 시각 문법은 아니다.

#### 보완 방향

1. 태그는 중립색 또는 단일 잉크블루로 통일한다.
2. 별점 대신 `입문`, `증명 중심`, `선행 글 필요`, `구현 포함` 같은 학습 방식 라벨을 사용한다.
3. 카테고리는 학습 장·시리즈로, 태그는 조밀한 색인으로 분리한다.
4. 정의·정리·증명·반례를 실제 주요 글에 점진적으로 적용한다.

### 4.2 엄밀한 학습 노트 정체성 — 7.9/10

#### 잘된 점

- 어두운 고정 사이드바가 종이색의 조용한 레일로 교체됐다.
- 화면의 중심이 프로필과 탐색 UI에서 글과 편집 구조로 이동했다.
- 종이색, 세리프 본문, 잉크블루, 어두운 코드 블록의 조합이 선명하다.
- 홈의 시간순 피드와 주제 통계는 축적된 학습 기록을 보여 준다.
- 섹션 번호와 단색 콜아웃이 "주장하는 종이"를 실제 화면에서 드러낸다.

#### 감점 근거

- 홈 추천 글의 둥근 카드와 장식용 원은 일반적인 랜딩 페이지 문법에 가깝다.
- 태그 페이지의 41개 동일 카드는 저자적 색인보다 자동 생성 목록처럼 보인다.
- 정리·증명 환경이 널리 쓰이지 않아 외형의 종이 분위기가 논증 구조보다 앞서는 구간이 있다.

#### 보완 방향

1. 추천 글 장식용 원을 시리즈 번호, 문서 번호, 증명 방식 같은 의미 있는 표식으로 교체한다.
2. 태그 페이지를 논문 색인처럼 재구성한다.
3. 정의·정리·증명·반례 문법을 실제 글에서 일관되게 사용한다.

### 4.3 장식의 필연성과 절제 — 5.3/10

#### 잘된 점

- 배경 glow와 과도한 hover lift가 대부분 사라졌다.
- 진행선과 TOC의 빛나는 효과가 제거됐다.
- 색상 수가 줄면서 본문과 강조의 관계가 차분해졌다.
- 접힌 레일이 콘텐츠를 밀어내거나 화면을 지배하지 않는다.

#### 감점 근거

- 홈 추천 카드의 장식용 원에는 콘텐츠 의미가 없다.
- 추천 글, 카테고리, 태그, 이전·다음 글에 둥근 카드가 반복된다.
- 일부 fade-in, press scale, hover 테두리·outline shadow가 남아 있다.
- `DESIGN_DIRECTION.md`가 폐기 대상으로 명시한 정적이지 않은 장식이 일부 유지된다.

#### 보완 방향

1. 장식용 원과 일반 목록의 fade-in부터 제거한다.
2. 카드 대신 괘선, 들여쓰기, 번호, 문단 간격으로 정보를 묶는다.
3. 그림자는 검색 모달·이미지 확대처럼 실제로 떠 있는 요소에만 사용한다.
4. 둥근 카드는 추천 글처럼 실제로 닫힌 단위에만 제한한다.

### 4.4 저자적 불규칙성과 시스템 일관성 — 5.5/10

#### 잘된 점

- 종이색과 잉크블루라는 중심 규칙이 생겼다.
- 조용한 레일의 비대칭 구조는 의도적이며 고유하다.
- 홈의 피드, 추천 글, 주제 통계가 서로 다른 편집 리듬을 유지한다.
- 콜아웃의 색상 체계가 단순해졌다.

#### 감점 근거

- 홈·레일 태그는 단색인데 글 상세·목록 태그는 hash 기반 다색이다.
- 카테고리와 태그 페이지는 구조와 CSS가 거의 복제돼 있다.
- 잉크블루 시스템 안에 과거 주황·갈색 literal과 alias가 일부 남아 있다.
- 한글 UI 라벨 일부는 약 9–11px의 작은 mono 텍스트다.
- `Pretendard`, `JetBrains Mono`는 font stack에 있지만 배포가 명시적으로 보장되지 않는다.
- 문서의 `theme-color`가 종이 인터페이스와 다른 어두운 값으로 남아 있다.

#### 보완 방향

1. 색·폰트·radius·motion 토큰을 정리해 과거 스타일 잔재를 제거한다.
2. 카테고리와 태그에 서로 다른 정보 구조를 부여한다.
3. 한글 UI는 sans와 최소 12–13px 수준을 보장한다.
4. 예외적 디자인은 증명 우회, 반례, 핵심 의존성처럼 콘텐츠상의 이유가 있을 때만 허용한다.

### 4.5 장문 가독성·탐색 지속성 — 7.3/10

#### 잘된 점

- 데스크톱 본문 폭 `614px`가 안정적이다.
- 모바일 본문 `345.4px`, `16.2px / 29.16px` 조합이 읽기 편하다.
- 코드와 표는 자체 스크롤되어 가로 레이아웃을 깨뜨리지 않는다.
- 코드 블록 대비가 높고 수식·표·본문의 구분이 명확하다.
- 데스크톱 TOC와 진행선은 긴 글에 유용하다.
- 조용한 레일 덕분에 중간 폭의 본문 공간이 이전보다 개선됐다.

#### 감점 근거

- 모바일에서 우측 TOC가 사라지고 대체 목차가 없다.
- 1만 px가 넘는 모바일 글에서 현재 위치를 확인하기 어렵다.
- 768px 직전 화면에서는 본문, `240px` TOC, 레일이 경쟁할 수 있다.
- 접힌 레일의 텍스트는 hover 중심이며 `:focus-within` 확장이 없다.
- 접힌 레일 링크의 접근 가능한 이름, dialog/drawer의 focus 관리가 충분하지 않다.
- 모바일 제목 `21px`은 긴 본문에 비해 존재감이 약하다.

#### 보완 방향

1. 글 상단에 접을 수 있는 모바일 TOC를 제공한다.
2. 중간 화면에서는 우측 TOC를 인라인 목차로 전환한다.
3. 접힌 레일은 `:focus-within`에서도 확장하고 각 링크에 명확한 이름을 제공한다.
4. 모바일 제목과 주요 H2의 크기 차이를 조금 더 벌린다.

| glow·장식 원·hover lift·fade | **부분 해결** | glow와 lift는 대부분 제거, 원·fade·일부 transform 유지 |
| 다색·과잉 콜아웃 | **부분 해결** | 3색 의미 체계로 축소했지만 박스 사용량은 많음 |
| 별점 난이도 | **미해결** | `★☆☆☆☆`–`★★★★★` 유지 |
| 모바일 TOC 부재 | **미해결** | 우측 패널 숨김, 대체 TOC 없음 |
| 중간 폭의 좁은 본문 | **부분 해결** | 조용한 레일로 본문 폭 확대, 고정 TOC 경쟁은 남음 |
| 작은 mono 한글 라벨 | **미해결** | 0.60–0.78rem 한글·혼합 라벨 다수 유지 |

완전 해결 1개, 부분 해결 3개, 미해결 5개다. 그러나 해결된 사이드바가 전체 인상을 크게 좌우하고, 색·콜아웃 정리가 여러 페이지에 동시에 영향을 주기 때문에 종합 점수는 0.6점 상승했다.

## 8. 다음 재평가 조건

다음 디자인 재평가에서도 같은 점수 척도와 가중치를 사용한다. 최소한 다음 화면을 같은 뷰포트에서 확인한다.

- `/` — `1040 × 650`
- `/categories/` — `1040 × 650`
- `/tags/` — `1040 × 650`
- `/blog/closest-pair-3/` — `1040 × 650`, `390 × 844`

재평가 시 다음 수치를 함께 기록한다.

- 레일 폭과 메인 콘텐츠 폭
- 본문 font-size, line-height, measure
- 태그 색인 항목 수와 전체 페이지 높이
- 모바일 장문 높이와 TOC 제공 여부
- code/table/math 가로 overflow 여부
- `prefers-reduced-motion`, keyboard focus, collapsed rail의 접근 가능한 이름

이 기준을 유지해야 이후 점수 변화가 단순한 인상 차이가 아니라 실제 디자인 시스템의 개선으로 해석될 수 있다.
