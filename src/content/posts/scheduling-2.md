## 구간 스케줄링 문제

[1편](/blog/scheduling-1)에서는 마감 기한과 **이익**이 있는 일들을 시간 슬롯에 배치해 이익의 합을 최대로 만들었다. 이번 문제는 다르다.

$N$개의 할 일 $J_1, \cdots, J_N$ 이 있다. 각각의 일은 두 값으로 이루어진다.

$$
J_i = (S_i, T_i)
$$

- $S_i$ : **시작 시간(start)** — 이 시각에 일이 시작된다.
- $T_i$ : **종료 시간(end)** — 이 시각에 일이 끝난다.

1편과 달리 일들은 길이가 제각각인 **구간(interval)** 이고, **이익은 모두 동일**하다. 한 번에 하나의 일만 할 수 있으므로, 시간이 겹치는 두 일은 함께 할 수 없다. 이익이 같으니 목표는 단순하다 — **겹치지 않게 고른 일의 개수를 최대로** 만드는 것이다.

여기서 두 일이 **겹친다**는 것은 한쪽이 끝나기 전에 다른 쪽이 시작한다는 뜻이다. 즉 $J_a$ 와 $J_b$ 는 $S_a < T_b$ 이고 $S_b < T_a$ 일 때 겹친다.

### 예시

다섯 개의 일이 있다고 하자.

$$
A=(1,3),\quad B=(2,5),\quad C=(4,7),\quad D=(6,8),\quad E=(8,10)
$$

$A$ 와 $B$ 는 시간 $2{\sim}3$ 에서 겹치고, $C$ 와 $D$ 는 $6{\sim}7$ 에서 겹친다. 겹치지 않게 고를 수 있는 가장 큰 묶음은 $\{A, C, E\}$ 로 **세 개**다. $B$ 나 $D$ 를 끼우려 하면 반드시 다른 일과 충돌해, 어떻게 골라도 세 개를 넘길 수 없다.

## Greedy 전략

선택 규칙은 한 문장이다.

1. **종료 시간이 빠른 일부터** 차례로 본다.
2. 지금 보는 일이 **이미 고른 일들과 겹치지 않으면** 고르고, 겹치면 버린다.

왜 하필 종료 시간일까. 빨리 끝나는 일을 고를수록 **뒤에 남는 시간이 많아져**, 그만큼 더 많은 일을 넣을 여지가 생기기 때문이다. 시작 시간이 빠른 일이나 길이가 짧은 일을 먼저 고르는 규칙은 이 직관을 보장하지 못한다 — 시작은 일러도 한참 뒤에 끝나는 일을 고르면 뒤가 모두 막힌다.

![다섯 구간 {A(1,3), B(2,5), C(4,7), D(6,8), E(8,10)}을 종료 시간 순으로 보며 greedy 선택 — A·C·E 채택, B·D는 앞선 일과 겹쳐 버림](/images/scheduling/intervals.svg)

위 예시에 규칙을 적용해 보자. 종료 시간 순서는 $A(3) \to B(5) \to C(7) \to D(8) \to E(10)$ 이다.

- $A=(1,3)$ : 처음이라 그냥 고른다.
- $B=(2,5)$ : $A$ 가 끝나는 시각 3 전인 2에 시작하므로 겹친다. 버린다.
- $C=(4,7)$ : 마지막으로 고른 $A$ 가 3에 끝났고 $C$ 는 4에 시작하니 겹치지 않는다. 고른다.
- $D=(6,8)$ : $C$ 가 7에 끝나기 전인 6에 시작하므로 겹친다. 버린다.
- $E=(8,10)$ : $C$ 가 7에 끝났고 $E$ 는 8에 시작하니 겹치지 않는다. 고른다.

결과는 $\{A, C, E\}$, 세 개로 최적과 일치한다. 직관은 그럴듯하지만, [그리디 알고리즘](/blog/greedy)에서 보았듯 국소 최적이 늘 전역 최적은 아니다. 증명이 필요하다.

## 정확성 증명

1편과 같은 도구, **교환 논증(exchange argument)** 을 쓴다. "우리 알고리즘의 부분 답을, 어떤 최적해로 손해 없이 맞춰 갈 수 있다"는 것을 단계마다 보인다.

### Invariant

일들을 종료 시간 순으로 정렬해 두고, $i$ 번째까지 처리한 greedy의 상태를 $A_i$ 라 하자. 다음을 불변식(invariant)으로 둔다.

> $A_i$ 의 모든 결정과 일치하는 **최적해 $S$ 가 적어도 하나 존재**한다. 즉 $j \le i$ 인 모든 $J_j$ 에 대해, $J_j$ 가 $A_i$ 에 채택되었는지 여부가 $S$ 에서도 똑같다.

이 불변식이 마지막($i = N$)까지 유지되면, $A_N$ 자체가 최적해 $S$ 와 채택 여부가 완전히 같으므로 $A_N$ 이 최적이다.

### Base

$i = 0$ 일 때는 아무 일도 처리하지 않은 상태다. $j \le 0$ 인 $j$ 가 없으므로 조건은 **공허하게 참(vacuously true)** 이다. 임의의 최적해를 $S$ 로 잡으면 된다.

### Step

$i$ 까지 불변식이 참이라 가정하고, $J_{i+1}$ 을 처리한 뒤에도 참임을 보인다. $A_i$ 와 $S$ 는 $j \le i$ 범위에서 채택 여부가 같다.

**경우 1: $A_i$ 가 $J_{i+1}$ 을 버리는 경우.**
$J_{i+1}$ 이 이미 고른 어떤 일 $J_k$($k \le i$)와 겹쳐서 버려진 경우다. $J_k$ 는 $A_i$ 에 채택되었고, $S$ 도 $j \le i$ 범위에서 일치하므로 $S$ 에도 $J_k$ 가 있다. 따라서 $S$ 역시 $J_{i+1}$ 을 가질 수 없다($J_k$ 와 겹치므로). 둘 다 $J_{i+1}$ 을 버리니 불변식이 유지된다.

**경우 2: $A_i$ 가 $J_{i+1}$ 을 고르는 경우.**
$S$ 가 $J_{i+1}$ 을 이미 갖고 있다면 채택 여부가 같아 그대로 유지된다. 문제는 **$S$ 에 $J_{i+1}$ 이 없는 경우**다. 이때 $S$ 를 손해 없이 고쳐 $J_{i+1}$ 을 넣을 수 있음을 보인다.

먼저, $S$ 에서 $J_{i+1}$ 과 겹치는 일은 **많아야 하나**다. $A_i$ 가 $J_{i+1}$ 을 골랐다는 것은 $J_{i+1}$ 이 $A_i$ 의 채택분(= $S$ 의 $j \le i$ 채택분)과 겹치지 않는다는 뜻이므로, $J_{i+1}$ 과 겹치는 $S$ 의 일은 모두 종료 시간이 $T_{i+1}$ 이상이다. 그런 일이 둘 있다면 둘 다 $T_{i+1}$ 직전 시각을 지나므로 서로도 겹쳐, $S$ 가 겹치지 않는 일들의 모음이라는 데 모순이다. 따라서 겹치는 일은 정확히 하나, 이를 $J_x$ 라 하자. 종료 시간 순 정렬에서 $x > i+1$ 이므로 $T_{i+1} \le T_x$ 다.

이제 $S$ 에서 $J_x$ 를 빼고 $J_{i+1}$ 을 넣는다. 개수는 그대로다. 새로 충돌이 생기지 않음은 $J_{i+1}$ 과 $J_x$ 의 시작 시간 관계로 확인한다.

- $S_x \le S_{i+1}$ 이면 $S_x \le S_{i+1} < T_{i+1} \le T_x$ 이므로 구간 $[S_{i+1}, T_{i+1})$ 은 $[S_x, T_x)$ 안에 들어간다. $J_x$ 와 겹치지 않던 일은 더 좁은 $J_{i+1}$ 과도 겹치지 않는다.
- $S_x > S_{i+1}$ 이면 $J_{i+1}$ 의 시작이 더 앞이다. 만약 $J_{i+1}$ 의 왼쪽에서 새로 겹치는 일 $J_y$ 가 $S$ 에 있었다면, $J_y$ 는 $J_x$ 와는 겹치지 않았으므로 $T_y \le S_x < T_{i+1}$ 이다. 그러면 $J_y$ 는 종료 시간이 $J_{i+1}$ 보다 빨라 $A_i$ 가 이미 처리한 일인데, $A_i$ 가 $J_{i+1}$ 을 골랐다는 것과 모순이다($J_y$ 가 채택분이면 겹쳐서 못 골랐을 것이다). 따라서 그런 $J_y$ 는 없고, 교환해도 충돌이 없다.

어느 쪽이든 $S$ 는 개수를 유지한 채 여전히 최적이며, 이제 $J_{i+1}$ 을 갖는다. 불변식은 $i+1$ 에서도 성립한다. $\blacksquare$

<div class="callout-title">증명의 뼈대</div>
<p>"greedy의 부분 답 $A_i$ 에 일치하는 최적해 $S$ 가 늘 존재한다"를 불변식으로 두고, 새 일 $J_{i+1}$ 을 처리할 때마다 $S$ 를 <strong>손해 없이 교환</strong>해 맞춘다. 핵심은 <strong>종료 시간 순</strong> 덕분에 "$S$ 에서 $J_{i+1}$ 과 겹치는 일 $J_x$ 는 많아야 하나이고 $T_{i+1} \le T_x$"가 보장된다는 것이다. 그래서 $J_x$ 를 더 일찍 끝나는 $J_{i+1}$ 로 바꿔도 개수가 줄지 않는다.</p>

<div class="callout callout-key">
<div class="callout-title">핵심 정리</div>
<ul>
<li><strong>구간 스케줄링</strong>은 시작 시간 $S_i$ 와 종료 시간 $T_i$ 를 가진 일들 중에서(이익은 모두 동일), 겹치지 않게 고른 일의 <strong>개수</strong>를 최대로 만드는 문제다.</li>
<li><strong>Greedy 전략</strong>은 <strong>종료 시간이 빠른 일부터</strong> 보고, 이미 고른 일과 겹치지 않으면 고른다. 빨리 끝낼수록 뒤에 더 많은 일을 넣을 여지가 남는다.</li>
<li>최적성은 <strong>교환 논증</strong>으로 증명한다. 종료 시간 순 정렬 덕에 "최적해에서 $J_{i+1}$ 과 겹치는 일은 많아야 하나이고 그 종료 시간이 $J_{i+1}$ 이상"이 보장되어, 손해 없는 교환이 가능하다.</li>
</ul>
</div>
