## 알고리즘의 아이디어

입력은 가중 그래프 $G = (V, E, w)$와 시작점 $v_0$, 정점 수는 $n$이다. **도착점은 따로 정하지 않는다.** 임의의 정점 $v_n$까지의 최단 경로를 구하면, 그 경로 위의 모든 중간 정점까지의 최단 거리도 동시에 얻어지기 때문이다 — 만약 중간의 어떤 부분 경로가 최단이 아니라면, 그것을 더 짧은 경로로 바꿔치기해 전체 길이를 줄일 수 있어 모순이 생긴다. 그래서 한 번에 모든 정점까지의 거리를 함께 구한다.

각 정점 $u$에 대해 **현재까지 추정한 최단 거리** $d_{min}(u)$를 관리한다. 초기값은 다음과 같다.

- 엣지 $(v_0, u)$가 존재하면: $d_{min}(u) = w(v_0, u)$
- 엣지가 없으면: $d_{min}(u) = \infty$

그리고 **이미 답이 확정된 정점들의 집합** $R$을 도입한다. 처음에는 $R = \{v_0\}$이다 (시작점까지의 거리는 자명히 $0$).

이제 $R$이 모든 정점을 포함할 때까지 다음 3-step을 반복한다.

1. $R^C$ (아직 확정되지 않은 정점들) 중 **가장 작은 $d_{min}$을 가진 정점** $u$를 고른다.
2. $u$를 $R$에 추가한다 — 이 시점에 $d_{min}(u)$는 $u$의 진짜 최단거리로 확정된다 (왜 그런지는 아래 [정확성 증명](#정확성-증명)에서).
3. $u$에서 나가는 엣지로 $R^C$의 인접 정점 $v$들에 대해 $d_{min}(v) = \min(d_{min}(v),\ d_{min}(u) + w(u, v))$로 갱신한다.

---

## Pseudo-code

```cpp
// G = (V, E, w), 시작 정점 v0
R       = { v0 }
d[v0]   = 0
d[v]    = ∞                    for all v ≠ v0
d[v]    = w(v0, v)             if (v0, v) ∈ E

while (R != V) {
    u = argmin { d[u] : u ∉ R };
    R = R ∪ { u };
    for each (u, v) ∈ E with v ∉ R {
        d[v] = min(d[v], d[u] + w(u, v));
    }
}
return d;
```

한 번의 반복은 **정점 하나를 $R$에 추가**한다. 시작 시 $|R| = 1$이고 끝났을 때 $|R| = n$이므로, 반복은 정확히 $n - 1$번 일어난다.

이 글은 다음 두 가정 위에서 동작한다 — (i) 모든 엣지 가중치는 $0$ 이상, (ii) 시작점 $v_0$에서 모든 정점에 도달 가능. 음의 가중치/도달 불가 정점 처리는 후속 글에서 다룬다.

## 단계별 진행 시각화

작은 그래프 한 개로 알고리즘을 따라가 보자.

- **정점:** $A, B, C, D, E$ (시작점 $A$)
- **엣지:** $A{-}B(2),\ A{-}C(5),\ B{-}C(1),\ B{-}D(7),\ C{-}D(3),\ C{-}E(8),\ D{-}E(2)$

![Dijkstra 진행 4단계 — A에서 시작해 R을 키워가며 거리를 갱신](/images/dijkstra-1/progress.svg)

각 단계에서 선택된 정점은 빨간색으로 채워지고, 그 정점이 갱신한 엣지는 굵은 빨간색이 된다. 단계별 변화를 표로 정리하면 다음과 같다.

| 반복 | 선택된 $u$ | $R$ (추가 후) | 갱신된 $d_{min}$ 값들 |
|---|---|---|---|
| 1 | $B$ | $\{A, B\}$ | $d[C]: 5 \to 3$, $d[D]: \infty \to 9$ |
| 2 | $C$ | $\{A, B, C\}$ | $d[D]: 9 \to 6$, $d[E]: \infty \to 11$ |
| 3 | $D$ | $\{A, B, C, D\}$ | $d[E]: 11 \to 8$ |
| 4 | $E$ | $\{A, B, C, D, E\}$ | — |

이 그림이 시사하듯이, Dijkstra는 항상 **가까운 노드부터** 추가한다 — $d_{min}$ 값이 작은 순서대로 $R$에 들어간다는 뜻이다.[^1]

[^1]: 이 사실의 정식 증명은 후속 글에서 다룬다 — 알고리즘이 추가하는 순서대로 $d_{min}$이 단조증가함을 보이면 된다.
