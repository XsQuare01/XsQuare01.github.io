## 알고리즘의 아이디어

Prim 알고리즘은 다음과 같이 동작한다.

1. **임의의 한 정점**에서 시작한다. 그 정점 하나만으로 된 트리가 초기 상태다.
2. **현재 트리에 인접한 간선** 중 **가장 작은 가중치**의 간선을 선택해 트리에 추가한다.
   - 단, cycle이 생기면 안 된다. 양 끝점이 모두 이미 트리에 있는 간선은 후보에서 제외한다.
3. 트리가 모든 정점을 포함할 때까지 2를 반복한다.

시작점은 어디라도 상관없다. MST는 모든 정점을 포함해야 하므로, 어느 정점에서 출발하든 결국 그 정점이 트리에 들어가기 때문이다. 시작점에 인접한 간선들 중 가장 작은 것부터 골라 트리를 키운다는 직관도 자연스럽다.

![Prim 알고리즘 진행 과정 — A에서 시작해 트리를 4단계에 걸쳐 키움](/images/prim/progress.svg)

각 단계에서 새로 추가된 정점은 빨간색으로 채워지고, 선택된 간선은 굵은 빨간 선이 된다. $|V| = 4$이므로 3개의 간선이 추가되면 종료된다.

## Pseudo-code

```cpp
// G = (V, E, w), 시작 정점 s
T   = {}            // 선택된 간선 집합
V_T = { s }         // 현재 트리에 포함된 정점 집합

while (V_T != V) {
    // V_T와 V \ V_T 사이를 잇는 간선 중 가장 작은 weight 선택
    (u, v) = argmin {
        w(e) : e = (u, v), u in V_T, v not in V_T
    };
    T   = T ∪ { (u, v) };
    V_T = V_T ∪ { v };
}
return T;
```

각 반복은 **정점 하나를 트리에 추가**한다. 시작 시 정점이 1개이고 끝났을 때 $n$개이므로, 반복은 정확히 $n - 1$번 일어난다. 따라서 선택된 간선도 $n - 1$개로, [트리의 간선 수](/blog/mst#정확히-n-1개의-간선)와 일치한다.

## Prim은 어떤 MST를 찾는가

[MST는 유일하지 않을 수 있다](/blog/mst#mst는-유일하지-않을-수-있다)는 것을 다시 떠올리자. 가중치가 같은 간선이 여러 개라면 동률(tie)을 어떻게 깨느냐에 따라 다른 MST가 나온다.

![Prim의 tie-breaking에 따라 가능한 두 MST](/images/prim/multiple-mst.svg)

Prim 알고리즘은 그중 **하나의 MST**를 찾는다. 어떤 간선이 선택되는지는 구현이 동률을 어떻게 깨는지에 따라 다르다. 정리하면:

- **MST가 유일하다면** Prim은 그 유일한 답을 찾는다.
- **MST가 여러 개라면** Prim은 그중 하나를 찾는다 — 어느 것이든 똑같이 정답이다.

---

## 정확성 증명

Prim 알고리즘이 항상 MST를 출력함을 증명한다. 핵심은 다음 **불변식**이다.

> **불변식:** Prim 알고리즘이 진행되는 도중의 간선 집합 $T_p$는 항상 어떤 MST $T_{mst}$의 부분집합이다. 즉, $T_p \subseteq T_{mst}$인 MST가 적어도 하나 존재한다.

알고리즘이 종료할 때 $|T_p| = n - 1 = |T_{mst}|$이므로, 부분집합 관계와 크기가 같다는 사실로부터 $T_p = T_{mst}$. 따라서 Prim의 출력 $T_p$는 MST다.

### Base — $|T_p| = 0$

처음에는 $T_p = \emptyset$. 공집합은 임의의 MST의 부분집합이다. 성립.

### Step — $|T_p| = k$에서 $|T_p| = k + 1$로

귀납 가정: $T_p \subseteq T_{mst}$인 MST $T_{mst}$가 존재. 알고리즘이 다음 간선 $e$를 선택해 $T_p' = T_p \cup \{e\}$로 확장했다고 하자. 두 경우로 나눈다.

#### 경우 1: $e \in T_{mst}$

이미 $T_p \subseteq T_{mst}$이고 $e \in T_{mst}$이므로 $T_p' \subseteq T_{mst}$. 불변식이 그대로 유지된다.

#### 경우 2: $e \notin T_{mst}$

이 경우, $T_p'$는 $T_{mst}$의 부분집합이 아니다. 따라서 $T_p'$를 부분집합으로 가지는 **다른 MST**가 존재함을 보여야 한다.

$T_{mst}$는 모든 정점을 잇는 트리이므로, $e$의 두 끝점을 잇는 경로가 $T_{mst}$ 안에 이미 존재한다. 거기에 $e$를 추가하면 **cycle**이 생긴다.

![Prim 정확성 증명 — T_mst에 e를 추가하면 cycle, cycle 위에 또 다른 boundary 간선 e' 존재](/images/prim/proof-cycle.svg)

cycle 위에는 $e$ 말고도 적어도 하나의 다른 간선 $e'$이 있다. 이 $e'$은 다음을 만족하도록 고를 수 있다.

- $e'$의 한 끝점은 현재 트리 $V_{T_p}$ 안에 있고, 다른 끝점은 그 밖에 있다 — 즉 $e'$ 역시 **Prim의 다음 선택 후보**다.
- $e' \in T_{mst}$, 그리고 $e' \notin T_p$.

이제 $e$와 $e'$의 가중치를 비교한다.

| 경우 | 결론 |
|---|---|
| $w(e) < w(e')$ | $T_{mst}$에서 $e'$을 빼고 $e$를 넣으면 더 작은 비용 — $T_{mst}$가 MST라는 가정에 **모순** |
| $w(e) > w(e')$ | $e'$도 Prim의 후보였는데 더 무거운 $e$를 골랐다 — 알고리즘 정의에 **모순** |
| $w(e) = w(e')$ | $T_{mst}$에서 $e'$을 빼고 $e$를 넣어도 비용 동일. 이 새로운 트리도 MST다 |

세 경우 중 **$w(e) = w(e')$만 가능**하다. 이 경우 새로운 MST를 다음과 같이 잡는다.

$$T_{mst}' = (T_{mst} \setminus \{e'\}) \cup \{e\}$$

- $T_{mst}'$은 여전히 MST다 (같은 비용, 같은 정점, 트리 구조 유지).
- $T_p' \subseteq T_{mst}'$ (원래 $T_p \subseteq T_{mst}$였고, $e'$은 어차피 $T_p$에 없었으며, 이제 $e$도 $T_{mst}'$에 들어 있다).

따라서 불변식이 $T_{mst}'$에 대해 유지된다.

귀납에 의해 알고리즘 종료 시점까지 불변식이 유지되고, Prim의 출력은 항상 MST다.

<div class="callout-title">증명의 묘미</div>

Prim이 매 단계 "지금 가장 좋아 보이는 선택"을 한다는 점에서 [그리디 알고리즘](/blog/greedy)이다. 그리디는 일반적으로 전역 최적을 보장하지 않지만, MST에서는 통한다. 위 증명이 그 이유다 — 그리디가 고르는 간선은 동률이거나 더 가벼울 수밖에 없고, 어느 경우든 어떤 MST에는 반드시 들어가도록 강제되기 때문이다.

</div>

<div class="callout callout-key">
<div class="callout-title">핵심 정리</div>

- Prim 알고리즘은 임의의 정점에서 시작해, 현재 트리에 인접한 간선 중 **가장 가벼운 것**을 반복적으로 추가하여 MST를 만든다.
- 매 반복마다 정점 하나가 트리에 추가되므로, 정확히 $n - 1$번 반복한 뒤 종료한다.
- 같은 가중치 간선이 여러 개 있어 MST가 여럿일 때, Prim은 **그중 하나**를 출력한다 — 모두 정답이다.
- 정확성은 불변식 "$T_p \subseteq T_{mst}$인 MST가 존재"로 증명된다. cycle 위의 대체 간선 $e'$과의 비교를 통해, 알고리즘이 어떤 간선을 골라도 항상 어떤 MST에 들어간다는 것을 보인다.
- 증명의 핵심: 세 가지 가중치 비교 경우 중 $w(e) < w(e')$과 $w(e) > w(e')$은 각각 MST 정의와 알고리즘 정의에 모순이므로, $w(e) = w(e')$만 가능 — 따라서 항상 정답으로 가는 길이 보장된다.

</div>
