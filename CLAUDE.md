# XsQuare01.github.io — 블로그 프로젝트 지침

Astro 기반 개인 기술 블로그. 포스트는 `src/content/posts/*.md`, 이미지는 `public/images/<slug>/`.

## 포스트 원본 자료 — 노션에서 읽어온다

이 블로그의 포스트는 사용자의 **노션(Notion)** 학습 정리를 바탕으로 작성한다. Notion MCP에 접근할 수 있다.

- "○○ 포스트를 작성해줘" 라는 요청을 받으면, **먼저 노션에서 해당 주제 페이지를 검색**한다.
  - `mcp__claude_ai_Notion__notion-search` 로 제목/키워드 검색 → `mcp__claude_ai_Notion__notion-fetch` 로 페이지 내용 가져오기.
  - 대부분의 알고리즘·전공 자료는 노션의 **"알고리즘 수업 정리"** / **"전공 과목 페이지"** 하위에 있다.
- **포스트 내용은 가져온 노션 자료의 범위 안에서만 작성한다.** 자료에 없는 내용을 자의로 추가하지 않는다. 보충이 필요하면 먼저 사용자에게 묻는다.
- 같은 주제가 여러 편(1편, 2편 …)으로 나뉘어 있으면, 노션의 해당 편 페이지 하나가 그 편 포스트 하나에 대응한다.

## 작성 규칙 (사용자 피드백 누적)

- **카테고리는 사용자가 직접 지정.** 미명시 시 작성 전에 반드시 물어본다. 기존 카테고리: `algorithm`, `theory`, `cryptography`.
- **날짜는 작성 당일 기준**으로 하드코딩하지 않고 오늘 날짜를 쓴다.
- **이모지(✅/❌) 대신 단순 기호/텍스트**(O/X, "버림" 등)를 쓴다.
- **수식**: 인라인은 `$...$`, 여러 줄·aligned 등 display 수식은 `$$`를 자체 줄에 둔다(KaTeX).
- **포스트 말미 callout 순서**: `callout-key`(핵심 정리) → `callout`(다음 포스트).
- **새 포스트 발행 시**, 시리즈상 이전 포스트 말미에 이 포스트를 가리키는 "다음 포스트" callout을 추가한다.

## SVG 다이어그램 규칙

- 배경 `#0f1117`, 제목 `#e2e8f0`, 부제 `#64748b`. 노드 fill `#1e3a5f` / stroke `#3b82f6`, 성공/배치 녹색 계열, 강조 주황 계열, 버림/오류 적색 계열.
- **세로 높이를 넉넉하게** 잡는다. 상단 padding ≥ 30px, 하단 ≥ 25px, 요소 겹침/클리핑 방지.
